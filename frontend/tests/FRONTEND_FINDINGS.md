# 前端风险修复记录

最近一次运行：

```bash
cd frontend
npm test
```

结果：34 个前端测试全部通过。以下问题已修复并由 Vitest 覆盖。

## 登录 / 注册

- 登录表单会 trim 虚拟地址并拒绝空白输入。
- 注册密码要求至少 6 位且同时包含数字和字母。
- 密码输入框文案已同步为“至少6位且包含数字和字母”。

## 路由 / 会话

- 未登录访问 dashboard 会跳回 `/login`。
- 已登录用户访问 `/login` 会被重定向到 `/dashboard/home`。
- JWT 注入、401 清理 token 的测试已通过。

## 首页

- “质押数量”“活跃借贷”改为读取质押/借贷接口，不再硬编码 `0`。

## 质押

- 前端预计可借额度已按资产 LTV 计算：ETH 80%、BTC 75%、USDT 90%。
- 质押数量为零或负数时按钮禁用，提交函数也会兜底拦截。

## 借贷

- 借贷金额为零、负数或超过可借额度时按钮禁用。
- 可借额度合计、预计本息计算测试已通过。

## 还款

- 还款金额为零、负数或超过剩余本金时按钮禁用。
- 只展示未还清贷款的测试已通过。

## 清算模拟

- 点击“应用价格并刷新风险”时会把用户实际调整过的 `tempPrice` 传给价格模拟接口。
- 高风险仓位才显示清算按钮的测试已通过。

## 数据中心

- 图表会校验 `dates.length === prices.length`，历史价格数据错位时不渲染风险图表。
- 统计格式化和正常价格历史渲染测试已通过。

## 供应链

`axios` 已升级到 `^1.16.1`，`npm audit --audit-level=high` 结果为 0 个漏洞。
