# 前端测试板块说明

## 测试目标

前端不是只测“页面能打开”，而是要暴露真实 DeFi 系统里容易导致资金误操作、风险误判或安全事故的问题：

- 钱包/登录状态是否被正确保护。
- JWT 是否被自动带入请求，401 是否清理会话。
- 金额、期限、空白字符串、超额操作是否在前端阻断。
- 前端展示的可借额度、利率、本息、清算风险是否与后端规则一致。
- 用户手动模拟价格时，是否真的应用了用户选择的价格。
- 图表输入数据是否经过长度和数值校验。
- 供应链依赖是否存在高危漏洞。

## 分层

- `api-contract.spec.js`：API 封装端点、参数和请求拦截器。
- `store-and-router.spec.js`：Pinia 登录态、JWT 路由保护、401 处理。
- `login-view.spec.js`：登录/注册表单校验。
- `defi-transaction-views.spec.js`：质押、借贷、还款、清算、图表组件。
- `static-defi-standards.spec.js`：静态规则检查，防止硬编码金融指标和危险模式。

## 真实 DeFi 前端重点

- 前端显示值不能和协议实际规则冲突，例如 LTV、清算线、剩余本金。
- 所有资金操作必须先在 UI 层拒绝明显非法输入，后端再做最终校验。
- 模拟/清算工具必须可复现，不能把用户设置的价格滑块替换成随机波动。
- 图表和风险提示必须验证输入数据形状，避免错误可视化误导用户。
- 已登录用户、未登录用户和失效 token 都要有明确路由行为。

## 运行方式

```bash
cd frontend
npm test
npm run test:watch
npm audit --audit-level=high
```

这些测试包含 regression 用例，当前预期会失败，用来先暴露问题。后续修复前端代码后，它们应变成回归保护。
