# Playwright 浏览器风险修复记录

最近一次运行：

```bash
cd frontend
npm run test:e2e -- --workers=1
```

结果：14 个浏览器测试全部通过。以下问题已在真实 Chromium 中验证修复。

本套测试使用 mock API，只验证浏览器中的真实页面交互，不依赖 Flask 后端。

## 预期关注点

- 未登录/已登录路由行为。
- 登录、注册表单真实输入。
- 质押、借贷、还款等资金操作按钮是否阻断明显非法输入。
- 前端展示的 LTV、本息、风险按钮是否和协议规则一致。
- 清算模拟滑块是否真的影响后续风险刷新。
- 数据中心图表是否拒绝错位数据。

## 已修复的问题

- 空白登录地址已被前端拦截。
- 弱注册密码已被前端拦截。
- 已登录用户访问 `/login` 会自动回 dashboard。
- 首页仓位统计不再硬编码为 `0`。
- 数据中心历史数据长度不一致时不渲染图表。
- 质押预估 LTV 已按 ETH/USDT/BTC 协议规则计算。
- 负数质押、0 借款、超额借款、超额还款已被按钮状态阻断。
- 清算滑块价格会作为确定性价格传给前端价格模拟请求。

## 已通过的浏览器基线

- 未登录访问 `/dashboard/home` 会跳转到 `/login`。
- 登录成功后会写入 token 并进入 dashboard。
- 数据中心正常统计值和正常价格曲线可以渲染。
- 清算列表只对高风险仓位显示“执行清算”按钮。
