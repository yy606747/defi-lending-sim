# DeFi Lending Sim 测试板块说明

## 测试依据

- `金融软工实验课12`：pytest、`pytest.raises`、参数化、fixture、monkeypatch、金额精度、边界、滑点/不变量、清算边界、覆盖率和 CI。
- `金融软件工程（第8讲）测试基础.pdf`：测试流程、单元/集成/系统/场景测试、自动化测试、回归测试。
- `金融软件工程（第8讲）白盒黑盒测试 .pdf`：程序单元测试覆盖执行路径、错误处理、模块接口、边界条件；黑盒测试使用等价类、边界值、错误推测、因果图；白盒测试关注语句/分支/条件/路径覆盖。

## 测试分层

- 单元测试：Service 函数、利率计算、价格仿真、金额计算、清算风险计算。
- 集成测试：Flask API、JWT 鉴权、注册登录、质押、借贷、还款、清算、统计。
- 场景测试：完整用户借贷流程、清算触发流程、失败请求不改变数据库状态。
- 回归测试：已观察或高概率缺陷，包括非法金额未捕获、USDT 价格历史长度不一致、边界舍入误判、超额还款、非法期限。

## 黑盒用例设计

- 等价类：合法金额、0、负数、非数字字符串、缺失字段、未知资产、未知用户、未授权请求。
- 边界值：`amount == 0`、`amount > 0` 最小正数、`loan_term == 30/31/90/91`、清算率刚好等于阈值、刚低于阈值、刚高于阈值。
- 错误推测：重复注册、借款超过可借额度、失败借款后额度被错误扣减、还款超过剩余本金、价格历史数组长度不匹配。
- 因果组合：有无活跃质押、有无未还贷款、质押是否属于当前用户、质押状态是否 active，共同决定是否允许解锁或清算。

## 白盒覆盖目标

- 用户模块：注册成功/重复、登录成功/失败、JWT 保护、信息更新。
- 资产模块：资产查询、随机价格模拟、稳定币价格边界、历史价格生成。
- 质押模块：资产存在性、金额合法性、LTV 计算、解锁状态和债务约束。
- 借贷模块：额度检查、跨质押扣减、期限利率分支、失败请求回滚。
- 还款模块：部分还款、还清、提前/到期类型、非法金额、非本人贷款。
- 清算模块：风险等级、动态清算阈值、严格边界、执行清算后的状态不变量。
- 仿真统计：用户数、质押价值、未还借款、清算次数、利用率、动态平均利率。

## 运行方式

```bash
python3 -m pytest
python3 -m pytest tests/backend/test_asset_and_simulation.py
python3 -m pytest -m financial
python3 -m pytest --cov=backend/app --cov-report=term-missing
```

这些测试刻意包含一部分“应当失败”的用例，用来暴露当前实现中的金融边界和输入校验问题。后续修复业务代码后，这些用例应变为通过的回归测试。
